Complete walkthrough — zero to app

Build Your EventPay App
From Zero to Live

Follow every step in order. By the end, you will have a real app installed on your phone that you can share with others.

⏱ ~90 minutes total 💰 100% Free 📱 Works on Android & iPhone ☁️ Google Sheets backend
1
Set Up Google Sheets (Your Database)
~10 minutes • Do this first, everything else depends on it
1

Open Google Sheets

On your phone or computer:

  1. Open your browser and go to sheets.google.com
  2. Sign in with your Google account (Gmail)
  3. Click the big + button or "Blank" to create a new spreadsheet
  4. At the top, click "Untitled spreadsheet" and rename it to: EventPay Database
2

Create 4 Sheets (Tabs)

At the bottom of Google Sheets, you see a tab called "Sheet1". You need 4 tabs total.

  1. Right-click on Sheet1 → click Rename → type Payments → press Enter
  2. Click the + icon at the bottom left to add a new tab → rename it to Complaints
  3. Add another tab → rename to Settings
  4. Add another tab → rename to Admins
⚠️ The names must be exactly: Payments Complaints Settings Admins — spelling and capital letters matter!
3

Set Up "Payments" Sheet Headers

Click on the Payments tab. In Row 1, type these in cells A1 through L1:

RefIDDateTimeNameVillagePhoneAmountUTRStatusVerifiedByVerifiedAtNotes
4

Set Up "Complaints" Sheet Headers

Click the Complaints tab. In Row 1, type in A1 through K1:

A1B1C1D1E1F1G1H1I1J1K1
DateTimeNameVillagePhoneEmailComplaintAttachmentStatusAdminReplyRepliedAt
5

2
Set Up Google Apps Script (Your Free Backend/API)
~15 minutes • This connects your app to Google Sheets
1

Open Apps Script from your Spreadsheet

  1. Go back to your EventPay Database Google Sheet
  2. In the top menu, click Extensions
  3. Click Apps Script
  4. A new tab opens — this is your code editor
  5. You will see a file called Code.gs with some default code
  6. Select all that default code and delete it (Ctrl+A then Delete)
2

Paste the Backend Code

  1. Open the file Code.gs from your downloaded EventPay folder
  2. Select all text in it (Ctrl+A) and Copy (Ctrl+C)
  3. Go back to the Apps Script editor
  4. Click inside the empty editor area and Paste (Ctrl+V)
  5. Click the 💾 Save button (floppy disk icon) or press Ctrl+S
  6. Give the project a name if it asks: type EventPay Backend
⚠️ If it asks "Authorization required" when saving, that's normal. We'll authorize in the next step.
3

Deploy as Web App — This Creates Your API URL

  1. In Apps Script, click the blue Deploy button (top right)
  2. Click New deployment
  3. Click the gear ⚙️ icon next to "Select type" → choose Web app
  4. Fill the form:
    • Description: type EventPay v1
    • Execute as: select Me (your email)
    • Who has access: select Anyone
  5. Click Deploy
  6. Google will ask for permissions — click Authorize access
  7. Choose your Google account → click Allow
  8. After deploying, you will see a URL like:
    https://script.google.com/macros/s/AKfycb.../exec
  9. Copy this URL — you will need it in the next phase!
🔴 Important: If you ever edit Code.gs later, you must click Deploy → Manage deployments → Edit → New version → Deploy again for changes to take effect.
3
Edit Config File & Prepare Your App Files
~10 minutes • Connect everything together
1

Understand Your File Structure

Your EventPay folder should look like this:

📁 EventPay/
📄 index.html  ← Home / Registration page
 📄 status.html  ← Payment status check
 📄 complaint.html  ← Complaint form
 📄 admin-login.html  ← Admin sign in
 📄 admin.html  ← Admin dashboard
 📄 offline.html  ← Shown when no internet
 📄 shared.css  ← Styles for all pages
 📄 config.js  ← ⭐ YOU EDIT THIS FILE
 📄 manifest.json  ← PWA app settings
 📄 sw.js  ← Service worker (offline)
 📄 generate_icons.html  ← Icon maker tool
 📁 icons/  ← CREATE THIS FOLDER
 📄 icon-72.png
 📄 icon-96.png
 📄 icon-192.png
 📄 icon-512.png
 📄 ... (all sizes)
 📁 screenshots/  ← CREATE THIS FOLDER (optional)
🔵 Items in purple are new files. The icons/ folder you need to create manually.
2

Edit config.js — The Only File You Need to Change

Open config.js in any text editor (Notepad on Windows, TextEdit on Mac).

Find these lines and replace with your actual values:

// Line 5 — Paste your Apps Script URL here:
SCRIPT_URL: "https://script.google.com/macros/s/YOUR_SCRIPT_ID_HERE/exec",

// Line 8 — Your UPI ID (the one you use to receive money):
UPI_ID: "yourname@paytm",

// Line 11 — Your event name (shown on all pages):
EVENT_NAME: "Wedding of Ram & Sita",

// Line 14 — Your organization name (shown on receipt):
ORG_NAME: "Ram Family",
✅ Save config.js after editing. That's the only file you need to edit — all pages read from this one file.
3

Generate App Icons

  1. Open generate_icons.html in your browser (double-click it)
  2. Type your event short name (2-3 letters, e.g. EP or WED)
  3. Pick a background color (purple #6366f1 is default)
  4. Click "Generate All Icons"
  5. Click Download for each icon one by one
  6. In your EventPay folder, create a new folder called icons
  7. Move all downloaded icon files into the icons folder
  8. Make sure names match exactly: icon-72.png, icon-96.png, icon-128.png, icon-144.png, icon-152.png, icon-192.png, icon-384.png, icon-512.png
4
Host on GitHub Pages (Free HTTPS Hosting)
~20 minutes • This gives your app a real URL on the internet
1

Create a GitHub Account (if you don't have one)

  1. Go to github.com
  2. Click Sign up
  3. Enter your email, create a password, choose a username
  4. Verify your email
  5. Done — GitHub is free forever
2

Create a New Repository (This is Like a Folder on GitHub)

  1. After logging in to GitHub, click the + icon at top right
  2. Click New repository
  3. Fill the form:
    • Repository name: type eventpay (all lowercase, no spaces)
    • Description: type anything like Event Payment App
    • Choose Public (required for free GitHub Pages)
    • Check "Add a README file"
  4. Click Create repository
3

Upload Your Files to GitHub

  1. You are now on your repository page
  2. Click Add file → Upload files
  3. Drag and drop ALL files from your EventPay folder:
    • index.html, status.html, complaint.html
    • admin-login.html, admin.html, offline.html
    • shared.css, config.js, manifest.json, sw.js
    • generate_icons.html
  4. At the bottom, under "Commit changes", type: Initial upload
  5. Click Commit changes
  6. For the icons folder: GitHub doesn't let you upload folders directly in the UI. Do this:
    • Click Add file → Upload files again
    • Drag your icon files from the icons folder
    • Before committing, in the file name box, add icons/ prefix — GitHub will auto-create the folder
    • OR use GitHub Desktop app (see step below for easier method)
💡 Easier method: Download GitHub Desktop from desktop.github.com — it lets you drag and drop entire folders including subfolders at once.
4

Enable GitHub Pages

  1. On your repository page, click Settings (gear icon in top tabs)
  2. In the left sidebar, click Pages
  3. Under Source, click the dropdown and select Deploy from a branch
  4. Under Branch, select main and keep / (root)
  5. Click Save
  6. Wait 2-3 minutes, then refresh the page
  7. You will see a green box saying: "Your site is live at https://YOURUSERNAME.github.io/eventpay"
🎉 Your app is now live on the internet with HTTPS! Share this URL with anyone.
5
Install on Your Phone as an App
~5 minutes • Works on Android and iPhone
1

Android Phone (Chrome Browser)

  1. Open Chrome on your Android phone
  2. Go to your app URL: https://YOURUSERNAME.github.io/eventpay
  3. Wait for the page to fully load
  4. Chrome will show a small banner at the bottom: "Add EventPay to Home screen" — tap it
  5. If the banner doesn't appear automatically:
    • Tap the 3 dots menu ⋮ at the top right of Chrome
    • Tap "Add to Home screen"
    • Tap "Add" in the popup
  6. The EventPay icon will appear on your home screen!
  7. Tap it — it opens like a real app, no browser bar!
✅ On Android, the app works exactly like a Play Store app. Full screen, your icon, works offline for pages already visited.
2

iPhone (Safari Browser)

  1. Open Safari on iPhone (must be Safari, not Chrome)
  2. Go to your app URL
  3. Tap the Share button (the box with an arrow at the bottom)
  4. Scroll down and tap "Add to Home Screen"
  5. Change the name if you want → tap Add
  6. Done! The app icon appears on your iPhone home screen
3

Share With Others

Anyone can install your app. Just share the URL:

https://YOURUSERNAME.github.io/eventpay

They open it in Chrome (Android) or Safari (iPhone) and follow the same "Add to Home Screen" steps above.

📱 For your village events: you can share this link on WhatsApp. People don't need to install anything from Play Store — just open the link and tap "Add to Home Screen".
4

How to Update Your App Later

When you want to change anything (event name, UPI ID, etc.):

  1. Edit the file on your computer (e.g. config.js)
  2. Go to your GitHub repository
  3. Click on the file you changed
  4. Click the pencil ✏️ edit icon
  5. Make your changes
  6. Click Commit changes
  7. GitHub Pages auto-updates in 1-2 minutes
  8. Users' apps update automatically next time they open it with internet
